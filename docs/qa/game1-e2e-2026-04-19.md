## Checkpoint 1 — FAIL detaljer

### Hva skjedde

1. Navigerte til https://spillorama-system.onrender.com/web/ ✅
   - Shell lastet normalt (tittel: "SPILLORAMA", login-form synlig)
   - Screenshot: `screenshots-bin-662/01a-login-page.png`

2. Fylte inn credentials `balltest@spillorama.no` / `BallTest123!` og klikket "Logg inn"

3. Fikk feilmelding: **"Ugyldig e-post eller passord."**
   - Screenshot: `screenshots-bin-662/01b-login-FAIL.png`

### Diagnose (tatt før stopp)

Rå API-sjekk via browser console:

```js
fetch('/api/auth/login', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({email: 'balltest@spillorama.no', password: 'BallTest123!'}),
})
```

Response:
```json
{
  "loginStatus": 400,
  "loginBody": {
    "ok": false,
    "error": {
      "code": "INVALID_CREDENTIALS",
      "message": "Ugyldig e-post eller passord."
    }
  },
  "healthStatus": 200,
  "backendVersion": "cloudflare"
}
```

**Konklusjon:** Staging er oppe (`/health` = 200), backend behandler login-request (400 ikke 5xx), men avslår credentialene. Dette er en **data/seed-mangel**, ikke en infra-bug eller Game 1-regresjon.

### Rotsak (hypotese)

BIN-643 PR #230 introduserte `apps/backend/scripts/seed-test-users-hall.ts` som tilordner `hall_id` til test-brukere. Scriptet ble kjørt mot dev-DB under BIN-643-arbeidet men er **ikke del av staging-deploy-prosedyren**. Sannsynlig at `balltest@spillorama.no` enten mangler på staging-DB eller har annet passord.
